# TEM AMCE Executive Dashboard

A single-page dashboard showing how the TEM **AMCE** area (Analytics, Marketing,
Communications, Events) is performing against вЂ” and contributing to вЂ” TEM's three
areas of focus and the quarterly scorecard.

The defining design decision is to **match the infrastructure to the refresh
cadence.** This data changes quarterly (the formal scorecard) and at most monthly
(operational KPIs) вЂ” it is not real-time. So the dashboard is built as a
**static React SPA that reads versioned JSON files from a CDN/object store**, with
the JSON regenerated by a scheduled pipeline. No live database, no app server, no
per-request compute вЂ” the cheapest and easiest-to-secure way to serve data at this
cadence.

## Architecture

```
Source systems в”Ђв”Ђ> Scheduled refresh job (ETL) в”Ђв”Ђ> writes versioned JSON
   (warehouse,        (runs on close /                to object store / CDN
    CRM, etc.)          monthly cron)                       в”‚
                                                            в–ј
                                          Static React SPA (CDN-hosted)
                                          fetches JSON at runtime, renders
```

- The SPA compiles to static assets and is served from a CDN/internal static host.
- Data lives in versioned JSON (e.g. `public/data/2026-Q2.json`) plus a small
  `manifest.json` listing available periods. Refreshing numbers never requires a
  code release.
- RAG status is computed once in the pipeline so the deck, dashboard, and any
  export always agree.

See [`PRD.md` В§6](PRD.md) for the full data contract (TypeScript types).
